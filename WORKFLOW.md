# Magicframe 自媒体运营完整工作流

基于 pool.html 内容池的自媒体运营系统,结合现有 baoyu-skills 和对话式工作流。

## 🎯 完整工作流

```
选题 → 策划 → 创作 → 制作 → 审核 → 发布 → 追踪
```

---

## 📋 详细流程

### 1️⃣ 选题阶段

**目标**: 从 pool.html 内容池选择合适的选题

**操作方式**:
```
你: "帮我从内容池选一个小红书的情感主题选题"
我: [读取 pool.html → 分析 5 个卖点 → 推荐最佳选题]
```

**我会提供**:
- 卖点名称 (便利贴/AI创作/家庭日常/习惯养成/家庭美术馆)
- 主题类型 (情感/趣味)
- 适合的母题 (3个)
- 核心关键词
- 场景问题包
- 情绪标签

**输出**: 选题信息 (在对话中展示)

---

### 2️⃣ 策划阶段

**目标**: 生成完整的内容策划案

**操作方式**:
```
你: "为这个选题生成策划案"
我: [生成策划案并保存]
```

**我会生成**:
- 3-5 个标题方案
- 内容大纲 (开头/正文/结尾)
- 视觉建议 (画面、风格、色调)
- 适合平台 (小红书/抖音/公众号)
- 预期效果

**输出**: `content/briefs/YYYY-MM-DD-{topic}.md`

---

### 3️⃣ 创作阶段

**目标**: 根据策划案创作具体内容

**操作方式**:
```
你: "根据策划案写小红书文案"
我: [创作内容并保存]
```

**内容形式**:

**📱 小红书图文**:
- 标题 (吸引眼球)
- 正文 (3-5 段,每段 2-3 行)
- 标签 (#关键词)
- 互动引导

**🎬 视频脚本**:
- 分镜表 (画面 + 时长)
- 旁白文案
- 字幕文案
- BGM 建议

**📄 长文章**:
- 结构化内容
- 小标题
- 引用和案例

**输出**: `content/drafts/YYYY-MM-DD-{topic}.md`

---

### 4️⃣ 制作阶段

**目标**: 生成视觉素材

**可用工具**:

**生成小红书图片**:
```bash
/baoyu-xhs-images content/drafts/YYYY-MM-DD-topic.md
```

**我会帮你**:
- 生成视频拍摄清单
- 提供素材建议
- 规划视觉方案

**输出**: 
- 图片: `xhs-images/{topic}/`
- 拍摄清单: `content/drafts/YYYY-MM-DD-{topic}-shotlist.md`

---

### 5️⃣ 审核阶段

**目标**: 内容质量检查

**操作方式**:
```
你: "帮我审核这个内容"
我: [检查并给出建议]
```

**检查项**:
- ✅ 品牌调性一致性
- ✅ 敏感词检测
- ✅ 平台规则合规性
- ✅ 内容完整度
- ✅ 标题吸引力
- ✅ 视觉效果

**输出**: 审核报告 (在对话中展示)

---

### 6️⃣ 发布阶段

**目标**: 发布到各平台

**可用工具**:

**微信公众号**:
```bash
# 先转换为 HTML
/baoyu-markdown-to-html content/drafts/YYYY-MM-DD-topic.md

# 发布
/baoyu-post-to-wechat content/drafts/YYYY-MM-DD-topic.html
```

**微博**:
```bash
/baoyu-post-to-weibo content/drafts/YYYY-MM-DD-topic.md
```

**小红书/抖音**:
```
你: "生成小红书发布文案"
我: [生成发布文案,你手动复制发布]
```

**输出**: `content/published/YYYY-MM-DD-{topic}.md` (记录发布信息)

---

### 7️⃣ 追踪阶段

**目标**: 记录和分析数据

**操作方式**:
```
你: "记录发布数据: 小红书 https://xxx 点赞50 收藏30"
我: [记录到数据文件]
```

**我会帮你**:
- 记录发布链接
- 记录初始数据
- 定期更新数据
- 生成月度分析报告

**输出**: `data/analytics/YYYY-MM.md`
